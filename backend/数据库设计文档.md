# 数据库设计文档

## 📌 概述

本项目使用 **SQLite** 数据库进行数据持久化，采用 Flask-SQLAlchemy ORM 框架进行数据库操作。数据库文件存储在 `instance/app.db`。

---

### 7. `predictions` - 预测记录表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 主键ID |
| model_id | INTEGER | FOREIGN KEY | 关联模型ID |
| input_features | JSON | NOT NULL | 输入特征(JSON格式，包含所有工艺参数) |
| prediction | FLOAT | NOT NULL | 预测结果(0-1，缺陷概率) |
| confidence | FLOAT | | 置信度(0-1) |
| shap_values | JSON | | SHAP值(JSON格式，用于根因分析) |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引**: `model_id`, `created_at`

**用途**: 记录每次质量预测的历史数据，支持：
- 查看历史预测记录
- 基于历史数据进行根因分析（SHAP解释）
- 预测数据统计分析

**数据流向**:
1. 用户在前端输入工艺参数
2. 调用 `/api/v1/predict/` API 执行预测
3. 后端自动保存预测记录到本表
4. 前端从 `/api/v1/predict/history` 获取历史记录展示

## 📌 数据库使用说明

### 1. 数据库位置

```
backend/
└── instance/
    └── app.db    # SQLite数据库文件
```

### 2. 访问数据库

**使用命令行**:
```bash
sqlite3 instance/app.db
```

**使用可视化工具**（如 Navicat、DBeaver）:
1. 打开数据库连接
2. 选择 SQLite 类型
3. 选择 `instance/app.db` 文件

### 3. 数据导入

运行数据导入脚本：
```bash
cd backend
python import_all_data.py
```

### 4. 数据库初始化

在 Flask 应用启动时，会自动创建所有表（如果不存在）：
```python
# app/main.py
from app.models import db
app = create_app()
with app.app_context():
    db.create_all()  # 创建所有表
```

### 5. 常见SQL查询示例

**查询质量标签数据**:
```sql
SELECT batch_id, defect_type, quality_status 
FROM quality_label 
WHERE quality_status = 1;  -- 查询缺陷品
```

**统计合格/缺陷数量**:
```sql
SELECT quality_status, COUNT(*) as count 
FROM quality_label 
GROUP BY quality_status;
```

**关联查询工艺参数与质量标签**:
```sql
SELECT p.batch_id, p.heating_temperature, q.defect_type
FROM process_data p
JOIN quality_label q ON p.batch_id = q.batch_id
WHERE q.quality_status = 1;
```

## 📌 数据备份与恢复

### 备份数据库
```bash
sqlite3 instance/app.db ".backup backup/app_backup.db"
```

### 恢复数据库
```bash
sqlite3 instance/app.db ".restore backup/app_backup.db"
```

### 导出为SQL文件
```bash
sqlite3 instance/app.db ".dump" > backup/app_dump.sql
```

### 从SQL文件导入
```bash
sqlite3 instance/app.db < backup/app_dump.sql
```

## 📌 性能优化建议

1. **索引优化**: 对经常查询的字段创建索引
2. **分页查询**: 大数据量查询使用分页参数
3. **批量操作**: 大量数据导入使用批量提交
4. **定期清理**: 清理过期的预测记录和日志

## 📌 安全注意事项

1. 数据库文件应设置适当的读写权限
2. 不要将数据库文件提交到版本控制系统
3. 敏感数据应加密存储
4. 定期备份数据库
